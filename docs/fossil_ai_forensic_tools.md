# Filament Forensic AI Tools

This document describes the domain-specific AI tools available in the Fossil Knowledge Base when using the Filament extension.

## 🛠 Overview

Filament extends the Fossil AI Agent with custom tools designed for cold-case analysis. These tools allow the AI to query the semantic database directly using structured logic, bridging the gap between vector search and relational metadata.

## 🚀 Setup

1.  **Ingest Data**:
    ```bash
    python3 code/scripts/ingest_all_to_fossil.py
    ```
2.  **Configure Tools**:
    ```bash
    python3 code/scripts/configure_fossil_tools.py
    ```
3.  **Start UI**:
    ```bash
    fossil ui -R data/knowledge.fossil
    ```

## 📖 Available Tools

### 📊 `filament_stats`
Provides a summary of the current knowledge base.
- **Usage**: `"Calculate knowledge base stats"`
- **Returns**: Counts of Unidentified Human Remains (UHR), Missing Persons (MP), Reddit narratives, and AI Leads.

### 🔍 `filament_search_cases`
Performs a keyword search across all notes with optional type filtering.
- **Arguments**:
  - `query` (string, required): The search term.
  - `source_type` (string, optional): One of `unidentified`, `missing_person`, `reddit`, `lead`.
- **Usage**: `"Search for cases involving green backpacks in unidentified remains"`

### 📄 `filament_case_detail`
Retrieves the full narrative body and metadata for a specific case.
- **Arguments**:
  - `source_ref` (string, required): The case number (e.g., `UHR-123` or `MP-456`).
- **Usage**: `"Show me the full details for case UHR-2022-0015"`

### 💡 `filament_list_leads`
Lists the top forensic match candidates generated by the Filament AI pipeline.
- **Usage**: `"List the latest potential matches"`
- **Returns**: A table of leads sorted by "heat" (confidence).

### 🧐 `filament_lead_detail`
Deep-dives into a specific forensic lead, showing the reasoning and the linked cases.
- **Arguments**:
  - `source_ref` (string, required): The Reddit URL or lead identifier.
- **Usage**: `"Analyze the lead linking UHR-X to the Reddit post from 2021"`

## 🔗 How it Works

These tools are implemented using **Fossil TH1 Extensions**. The logic is stored in `cfg/filament-tools.th1` and registered at runtime when the Fossil repository is opened. This keeps the core Fossil binary clean and dependency-free while allowing Filament to provide deep forensic intelligence.
